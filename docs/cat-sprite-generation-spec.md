# Cat Sprite Generation Spec

This file records the target runtime assets for the photo-based cat sprite pass. Do not generate identity details until the source pet photo has been analyzed and `assets/references/cat-identity-lock.json` has been filled.

## Identity Lock

Before generating frames, inspect the source photo and lock these fields:

- Coat color
- Marking positions
- Eye color
- Tail color and shape
- Body type
- Facial features

User-provided appearance corrections are hard constraints and must be appended to `hardCorrections`.

## Required Action Folders

Each folder below must contain exactly 16 transparent PNG frames, named `<action>_0001.png` through `<action>_0016.png`.

- `idle`
- `walk`
- `walk_left`
- `sleep`
- `sleepy`
- `waking`
- `surprised`
- `shy`
- `dragging`
- `waving`
- `teaser`

Runtime mapping from the current MVP state names:

- `walking` uses `walk` or `walk_left` depending on direction.
- `sleeping` uses `sleep`.
- Mouse-over teaser interaction uses `teaser`.

## Frame Requirements

- 512x512 PNG.
- Transparent background with an alpha channel.
- Pet centered with consistent scale.
- No frame may touch or clip against the canvas edge.
- `walk` is a right-facing walk cycle.
- `walk_left` must be generated by horizontally mirroring `walk`, not by reversing frame order.
- `teaser` keeps the cat body at stable scale while the teaser toy moves.
- `sleep` uses natural breathing motion, preferably from the first frame with subtle body rise/fall.
- Final runtime frames must not use alpha-blended interpolation that leaves transparent-edge ghosting.
