# 桌宠猫咪视频生成提示词手册

本文档用于生成桌宠动作视频。默认角色参考为 `assets/origin/鱼仔参考图.png`，后续如果要换成其他猫，只需要替换参考图和猫咪特征描述，仍按同一套动作、规格和命名生成视频。

如果使用可灵 AI 批量生成视频，先阅读 `docs/kling-integration.md`，并检查 `docs/kling-action-generation-plan.json` 中的动作计划。

## 使用目标

- 所有桌宠动作视频都以同一张猫咪参考图为身份锚点。
- 视频用于转成透明 PNG 序列帧，因此必须稳定、清晰、可抠像。
- 日常动作和交互动作分开生成，方便后续插入播放、回切日常动作和统一调度。
- 每个动作都要能转换为 24 fps、72 帧的运行时序列帧。

## 可替换变量

生成其他猫咪时，优先替换下面这些变量，不要改动作结构。

| 变量 | 鱼仔默认值 | 换猫时填写 |
| --- | --- | --- |
| `{角色参考图}` | `assets/origin/鱼仔参考图.png` | 新猫咪参考图路径 |
| `{猫咪特征}` | 灰白双色短毛猫，圆脸，琥珀色眼睛，白色胸口和前爪，灰色背部和尾巴，安静可爱 | 新猫咪的毛色、体型、眼睛、斑纹、气质 |
| `{画面比例}` | 竖版 1080x1348 或接近 4:5 | 保持同一比例 |
| `{背景}` | 纯绿色绿幕背景 | 纯绿色绿幕背景 |
| `{时长}` | 3 秒 | 3 秒 |
| `{帧率}` | 24 fps | 24 fps |

## 统一视频规格

所有动作视频都必须遵守：

- 使用 `{角色参考图}` 作为强参考，猫咪身份、脸型、毛色、眼睛和身体比例保持一致。
- 单只猫，全身入镜，身体不要被裁切。
- 固定机位，镜头不推拉、不摇晃、不旋转。
- 猫咪位于画面中心，脚底和尾巴保留足够边距。
- 纯绿色绿幕背景，背景颜色稳定、无阴影干扰。
- 无水印、无 logo、无字幕、无文字、无 UI。
- 不要出现人手、道具、其他动物或多只猫。
- 动作幅度温和，适合桌宠循环播放。
- 视频时长 3 秒，24 fps，动作开始和结束姿势尽量接近，方便循环或回切。
- 光照柔和稳定，不要闪烁，不要强反光。
- 真实猫咪质感，毛发自然，但轮廓要清楚，方便抠绿。

## 通用正向提示词模板

```text
使用参考图 {角色参考图} 中的同一只猫作为唯一角色。猫咪特征为：{猫咪特征}。
生成一个桌面宠物动作视频，竖版 {画面比例}，时长 {时长}，{帧率}，固定机位，纯绿色绿幕背景。
猫咪全身入镜，居中，脚底和尾巴不要被裁切，动作自然、轻柔、可爱，适合转成透明序列帧循环播放。
保持猫咪的脸型、毛色、眼睛颜色、花纹位置、身体比例和毛发质感与参考图一致。
画面中只能有这一只猫，没有人、没有道具、没有文字、没有水印、没有 logo。
动作：{动作描述}
```

## 通用负向提示词

```text
不要改变猫咪品种、毛色、眼睛颜色、脸型和斑纹。
不要出现两只猫、多余肢体、畸形爪子、断尾、身体穿模、脸部变形。
不要出现人、手、玩具、家具、食盆、文字、水印、logo、边框、UI。
不要镜头运动、不要变焦、不要旋转、不要切镜头、不要背景变化。
不要裁切耳朵、尾巴、脚掌或身体边缘。
不要复杂阴影、不要反光地面、不要透明背景，背景必须是纯绿色绿幕。
不要大幅跳跃、快速奔跑、翻滚或剧烈动作。
```

## 命名与分类

源视频建议放在 `assets/origin`，文件名使用中文动作名，运行时 action 使用英文蛇形命名。

| 分类 | 源视频命名示例 | action 命名示例 |
| --- | --- | --- |
| 日常动作 | `鱼仔日常待机正面.mp4` | `idle_primary` |
| 交互动作 | `鱼仔交互前肢抬起.mp4` | `paw_raise` |
| 过渡动作 | `鱼仔过渡待机到前肢.mp4` | `idle_to_paw_raise` |

## 日常动作提示词

日常动作属于 `daily`。它们应该适合长时间播放，动作开始和结束姿势尽量接近。

### 1. 默认待机 `idle_primary`

```text
{通用正向提示词}
动作：猫咪正面坐着，保持安静自然的待机姿势，轻微呼吸起伏，偶尔眨眼，耳朵有非常轻微的自然抖动。动作幅度小，开始和结束姿势几乎一致，适合无缝循环。
{通用负向提示词}
```

### 2. 备用待机 `idle_secondary`

```text
{通用正向提示词}
动作：猫咪正面坐着，头部缓慢轻微左右转动，眼睛保持灵动，身体基本稳定，尾巴贴近身体轻微摆动。动作自然缓慢，开始和结束回到接近正面坐姿。
{通用负向提示词}
```

### 3. 摇尾待机 `daily_tail_wag`

```text
{通用正向提示词}
动作：猫咪坐姿不变，尾巴从身体一侧轻柔摆动，摆动节奏平稳，身体和头部保持稳定，偶尔眨眼。动作适合循环播放，开始和结束尾巴位置尽量接近。
{通用负向提示词}
```

### 4. 慵懒眨眼 `slow_blink`

```text
{通用正向提示词}
动作：猫咪保持坐姿，缓慢眨眼，表情放松，头部微微下沉再恢复，像在安静陪伴用户。动作很轻，不改变整体姿势，适合穿插在日常待机中。
{通用负向提示词}
```

### 5. 左右观察 `look_around`

```text
{通用正向提示词}
动作：猫咪保持坐姿，先缓慢看向左侧，再缓慢看向右侧，最后回到正面。身体稳定，只有头部和眼神移动，动作自然，不要突然转头。
{通用负向提示词}
```

### 6. 走路右向 `walk`

```text
{通用正向提示词}
动作：猫咪以轻松小步向画面右侧走动，身体完整入镜，步伐稳定，头部和尾巴自然跟随。动作节奏均匀，适合桌宠在桌面上移动。
{通用负向提示词}
```

说明：`walk_left` 优先由 `walk` 镜像生成，除非右向镜像后斑纹方向必须保持真实。

## 交互动作提示词

交互动作属于 `interactive`。它们可以幅度更明显，但必须能从待机姿势自然进入，并在结束时回到接近日常待机的姿势。

### 1. 前肢抬起 `paw_raise`

```text
{通用正向提示词}
动作：猫咪从正面坐姿开始，缓慢抬起一只前爪，像在向用户打招呼或回应鼠标靠近，停顿一小会儿，然后前爪自然放回地面，结束时回到接近初始坐姿。
{通用负向提示词}
```

### 2. 被点击惊讶 `click_surprised`

```text
{通用正向提示词}
动作：猫咪从坐姿开始，像被轻轻点到一样短暂睁大眼睛，耳朵轻微竖起，身体轻轻一震，然后恢复平静坐姿。动作可爱、轻微，不要害怕或夸张跳起。
{通用负向提示词}
```

### 3. 害羞低头 `shy`

```text
{通用正向提示词}
动作：猫咪从正面坐姿开始，轻轻低头，眼神变柔和，耳朵轻微向两侧放松，像害羞一样停顿，然后慢慢抬头恢复正面坐姿。
{通用负向提示词}
```

### 4. 被拖拽悬停 `dragging`

```text
{通用正向提示词}
动作：猫咪身体保持完整，四肢轻微收起，像被温柔托起一样有轻微悬浮感，尾巴自然下垂或轻摆，表情有点惊讶但可爱。动作幅度小，适合拖拽时循环。
{通用负向提示词}
```

### 5. 唤醒 `waking`

```text
{通用正向提示词}
动作：猫咪从困倦坐姿开始，眼睛半闭，轻轻抬头，慢慢睁眼，耳朵恢复竖起，最后回到清醒的正面坐姿。动作柔和，适合从睡眠状态恢复。
{通用负向提示词}
```

### 6. 伸懒腰 `stretch`

```text
{通用正向提示词}
动作：猫咪从坐姿开始，前爪向前轻轻伸展，背部微微拉伸，尾巴自然摆动，然后收回前爪恢复坐姿。动作温和，不要离开画面，不要趴得过低导致身体被裁切。
{通用负向提示词}
```

## 过渡动作提示词

过渡动作属于 `transition`。只有在日常动作和交互动作切换不自然时才需要生成。

### 1. 待机到前肢抬起 `idle_to_paw_raise`

```text
{通用正向提示词}
动作：猫咪从正面待机坐姿开始，身体重心轻微调整，肩部和前爪准备抬起，但不要完全抬爪。动作很短、很柔和，结束姿势要能自然衔接到抬前爪动作的第一帧。
{通用负向提示词}
```

### 2. 前肢抬起回待机 `paw_raise_to_idle`

```text
{通用正向提示词}
动作：猫咪从前爪刚放下的姿势开始，轻微调整身体和头部，回到稳定正面坐姿。动作很短，结束姿势要接近日常待机第一帧。
{通用负向提示词}
```

## 13 状态建议清单

后续补齐 13 种状态时，建议按下面分类生成。

| 状态 | 分类 | 建议 action | 是否循环 | 备注 |
| --- | --- | --- | --- | --- |
| 待机 | `daily` | `idle_primary` | 是 | 主循环 |
| 备用待机 | `daily` | `idle_secondary` | 是 | 日常变化 |
| 摇尾 | `daily` | `daily_tail_wag` | 是 | 可作为日常变化 |
| 眨眼 | `daily` | `slow_blink` | 否 | 可插入日常 |
| 左右观察 | `daily` | `look_around` | 否 | 可插入日常 |
| 右走 | `daily` | `walk` | 是 | 桌面移动 |
| 左走 | `daily` | `walk_left` | 是 | 优先镜像 |
| 鼠标靠近 | `interactive` | `paw_raise` | 否 | 结束回待机 |
| 点击惊讶 | `interactive` | `click_surprised` | 否 | 点击触发 |
| 害羞 | `interactive` | `shy` | 否 | 多次点击触发 |
| 拖拽 | `interactive` | `dragging` | 是 | 拖拽期间循环 |
| 唤醒 | `interactive` | `waking` | 否 | 从睡眠恢复 |
| 伸懒腰 | `interactive` | `stretch` | 否 | 定时或休息提醒触发 |

## 换成其他猫咪的流程

1. 准备新的猫咪参考图，放到 `assets/origin`。
2. 在本文档中替换 `{角色参考图}` 和 `{猫咪特征}`。
3. 按日常动作、交互动作、过渡动作分别生成视频。
4. 检查每个视频是否符合：纯绿幕、无水印、全身入镜、固定机位、3 秒、24 fps。
5. 将通过检查的视频放入 `assets/origin`。
6. 开始生成帧前，列出源文件、动作名称、分类、目标路径、验证方式，得到确认后再执行。
7. 执行序列帧生成与运行时校验。

## 质量验收清单

- 猫咪身份和参考图一致。
- 视频没有水印、文字、logo。
- 背景为稳定纯绿色绿幕。
- 全身、耳朵、尾巴、脚掌没有被裁切。
- 动作开始和结束姿势适合循环或回切。
- 动作节奏平稳，没有明显跳帧。
- 没有多余肢体、畸形爪子、脸部变形。
- 可转成 24 fps、72 帧运行时序列帧。
- 转换后通过 `npm run validate:runtime-animations`。
- 转换后通过 `npm run validate:animation-smoothness`。
